Budget Enforcement

Set spend limits per project, team, or any tag. Get alerts at threshold, or enforce hard limits that block calls before they're made.

Set a budget

import kostrack

kostrack.configure(dsn="postgresql://...")
enforcer = kostrack.budget_enforcer()

# Alert at 80%, hard limit at 100%
enforcer.set_budget(
    tag_key="project",
    tag_value="openmanagr",
    period="monthly",
    limit_usd=50.0,
    alert_threshold=0.80,
    enforce=False,   # alert only — set True to block calls
)

Alert-only mode

When enforce=False, Kostrack logs a warning when the threshold is crossed but never blocks calls:

# WARNING: Budget alert: project=openmanagr (monthly) at 83.2% ($41.60 / $50.00)

Hard enforcement

When enforce=True, calls raise BudgetExceededError when the limit is reached. Use guard() to wrap calls:

from kostrack.budget import BudgetExceededError

try:
    with enforcer.guard(tags={"project": "openmanagr"}):
        response = client.messages.create(...)
except BudgetExceededError as e:
    print(f"Blocked: {e}")
    # Handle gracefully — return cached result, use cheaper model, etc.

Check budget status

# Check all budgets matching these tags
status = enforcer.check(tags={"project": "openmanagr"})
for b in status:
    print(f"{b['tag_value']}: {b['pct']*100:.1f}% (${b['spent']:.4f} / ${b['limit']:.2f})")

# Get all budgets
all_budgets = enforcer.get_status()
Cache TTL

The enforcer caches budget state for 60 seconds by default. This means enforcement can lag by up to 60 seconds. Adjust with BudgetEnforcer(dsn=..., cache_ttl=30.0).
